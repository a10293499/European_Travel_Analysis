資料清整、圖表視覺化、Kmeans分群

資料

Note執行摘要：2026 歐洲旅遊市場數據洞察 (Executive Summary)

本報告針對歐洲 21 個國家、209 個熱門景點進行多維度分析，核心發現如下：

市場結構：人文城市為觀光主力，數據顯示超過6成的景點屬於「人文城市 (Urban & Culture)」。在時間分佈上，歐洲旅遊呈現明顯的雙峰趨勢，春季與秋季為最受推薦的走訪時機。

經濟地景：非歐元區的高消費陷阱與自然景點溢價，打破了「非歐元區較便宜」的直覺。受瑞士、北歐及冰島影響，非歐元區的高消費景點比例（64%）高於歐元區（56.9%）。此外，自然景觀與水岸海島景點有近 8 成屬於高消費區，顯示此類資源具備較高的市場溢價能力。

市場分群： 依景點特性並透過 K-Means 聚類演算法（解釋力 83.5%），將市場精準切分為四大族群：

精緻小眾秘境 (Cluster 1)：低人流、高消費、高安全性。

高 CP 值避境 (Cluster 2)：中人流、低消費、高安全性。

高貴明星景點 (Cluster 3)：高人流、高消費、高安全性。

一線大城戰區 (Cluster 4)：極高人流、高消費、具潛在風險。

資料清整

在本階段，我們處理原始資料中的編碼問題（latin1 轉 UTF-8）、修正欄位錯位，並將非數值字串（如遊客數、物價等級）轉換為可分析的格式。

實際處理如下：

  • 修復資料錯位，像是Category中出現如經緯度等數字

  • 統一標籤：

    1. 旅遊人數：統一成數字，若為範圍數值則取中間值
    2. 適合旅遊時間：統一為Spring/Summer/Fall/Winter/Year-round

  • 新增標籤：

    1. 貨幣Currency：歐元區/非歐元區

    2. 安全等級Safety：拆出安全等級(Safety Level)和需要注意的重點(Safety Notes)

    3. 景點類別Category：區分為五大項如下

      人文城市 Urban & Culture (涵蓋城市、歷史遺蹟、博物館)

      自然景觀 Nature & Landscapes (涵蓋山脈、森林、國家公園)

      水岸海島 Coastal & Islands (涵蓋海岸、海灘、島嶼)

      休閒娛樂 Leisure & Recreation (涵蓋主題樂園、花園、遊樂場)

      其他 Other

#定義清理函數與處理邏輯
clean_tourists <- function(x) {
  x <- str_to_lower(x)
  x <- str_replace_all(x, ",", "")
  
  # 處理區間類型 (如 35-40 million)
  if (str_detect(x, "-")) {
    nums <- as.numeric(str_extract_all(x, "[0-9.]+")[[1]])
    val <- mean(nums)
  } else {
    val <- as.numeric(str_extract(x, "[0-9.]+"))
  }
  
  # 處理 million 單位
  if (str_detect(x, "million")) {
    val <- val * 1000000
  }
  return(val)
}

# 3. 主清理流程
cleaned_df <- raw_data %>%
  #修復資料錯位: 
  mutate(across(everything(), as.character)) %>%
  rowwise() %>%
  mutate(
    is_displaced = !is.na(as.numeric(Category)),
    Region = if_else(is_displaced, "Unknown", Region),
  ) %>%
  
  #統一標籤
  mutate(
    Currency = str_replace(Currency, "Euro \\(EUR\\)", "Euro"),
    Language = str_replace(Language, " \\(and (.*?)\\)", ", \\1"),
    
    #簡化最佳旅遊時間
    Best_Time_Simplified = case_when(
      str_detect(`Best.Time.to.Visit`, "Year-round|anytime") ~ "Year-round",
      TRUE ~ {
        seasons <- c()
        if(str_detect(`Best.Time.to.Visit`, "Spring")) seasons <- c(seasons, "Spring")
        if(str_detect(`Best.Time.to.Visit`, "Summer")) seasons <- c(seasons, "Summer")
        if(str_detect(`Best.Time.to.Visit`, "Fall|Autumn")) seasons <- c(seasons, "Fall")
        if(str_detect(`Best.Time.to.Visit`, "Winter")) seasons <- c(seasons, "Winter")
        paste(seasons, collapse = ", ")
      }
    )
  ) %>%
  
  #轉換遊客數為數值
  mutate(
    Approximate_Annual_Tourists_Num = sapply(Approximate.Annual.Tourists, clean_tourists)
  ) %>%
  
  #修正特定字串亂碼 (手動修正已知歐洲特殊字元)
  mutate(across(where(is.character), ~str_replace_all(., "â€", "ö"))) %>%
  mutate(across(where(is.character), ~str_replace_all(., "â€ ", "å"))) %>%
  mutate(
    # 拆分出安全等級
    Safety_Level = str_split(Safety, " but") %>% map_chr(1) %>% str_trim() %>% str_replace(",$", ""),
    Safety_Notes = str_extract_all(Safety, "(?<=of\\s|for\\s)[\\w\\s]+(?=,|\\.|$)") %>%
      map_chr(~ {
        if (length(.x) == 0) return("None")
        clean_note <- str_replace_all(.x, "watch out |be aware |the ", "") %>%
          str_replace_all(" and ", ", ") %>%
          str_trim() %>%
          str_to_title()
        return(clean_note)
      })
  )%>%
  mutate(
    # 創建新類別欄位：Category_Group
    Category_Group = case_when(
      str_detect(Category, "City|Town|Village|District|Square|Museum|Castle|Fortress|Historical") ~ "Urban & Culture",
      str_detect(Category, "Mountain|Valley|Forest|National Park|Lake|Cave|Nature") ~ "Nature & Landscapes",
      str_detect(Category, "Coastal|Island|Beach") ~ "Coastal & Islands",
      str_detect(Category, "Park|Garden") ~ "Leisure & Recreation",
      TRUE ~ "Other"
    )
  )

遊客流量分析

遊客流量分布與熱門景點TOP10

約1/3的觀光地區每年的觀光人潮集中在50-200萬人(0.5-2M)。而遊客數最多的地點第一名為巴黎(Paris)，每年湧入3千8百萬人，遠遠高於第二名倫敦(2千5百萬人)，第三~十名則落在1千萬~1千5百萬人。

遊客流量分布圖右側的長尾代表了極端熱門的景點（如年遊客量超過 10M 的倫敦、巴黎、羅馬）。在分析中，雖然它們屬於 Outliers (離群值)，但因為它們代表了具備高度商業價值的「地標級」目的地，因此在此處將之保留下來。

生活消費等級

整體消費來看，大多集中在高或中度消費。以景點屬性區分消費水準(下左圖)，將近9成的自然景觀為高消費；其次，水岸海島也有8成左右屬於高消費，由於這些地方很多屬於度假勝地，在消費上通常會較高，或是國家公園的交通成本，以及可能含有門票費等因素造成消費水平較高。而休閒娛樂的景點數量不多，大部分為免費可入場的花園。

歐元區與非歐元區相比之下，非歐元區的生活消費較高，由於非歐元區涵蓋瑞士、北歐與冰島這些高消費的國家。

K-Means分群決定：手肘法(Elbow Method)

以手肘法(Elbow Method) 確認K-Means應該要分成多少族群，從結果來看，在k=4時，群內平方和(Within Sum of Square, WSS) 大幅度下降，代表分群效果顯著，分4群已可以抓到數據中的主要差異，再區分下去的效益並不高。

Tip什麼是「群內平方和 (Within Sum of Square, WSS)」?

對於某個群組 \(C\)，WSS 計算公式如下：

\[WSS = \sum_{i \in C} (x_i - \mu)^2\]

\(x_i\)：群組內的每一個資料點。 \(\mu\)：該群組的中心點（平均值）。

所以我們去計算每一個點和該群組中心點的距離，最後將所有群組（\(k=1 \dots 4\)）的 WSS 加起來，即為所求。

簡單翻譯WSS為「群組的擠迫程度」。也就是當 WSS 越小，代表同一群裡的資料點靠得越近（越像）。雖然WSS越小代表區分的越仔細，但需要注意和k之間的平衡，太多群會有過度擬合的問題，反而失去分類的意義。所以我們要找那個「手肘」，也就是當我增加群數時，WSS 卻不再大幅下降的那個平衡點

// 3. 核心處理與過濾邏輯
recommendations = {
  // A. 預處理：先把 1,2,3,4 標籤化，確保表格顯示不為空
  let processed = data.map(d => {
    let costText = "";
    // 對應你的分群數據特徵
    if (String(d.cluster) === "2") costText = "低消費";
    else if (String(d.cluster) === "1" || String(d.cluster) === "4") costText = "中消費";
    else costText = "高消費";
    
    return { ...d, Cost_Label: costText };
  });

  // B. 開始過濾
  return processed.filter(d => {
    // 1. 類別過濾 (必須精確符合)
    const matchCategory = d.Category_Group === category;
    
    // 2. 預算過濾 (全相容邏輯)
    let matchBudget = false;
    if (budget === "Low") {
      matchBudget = (String(d.cluster) === "2"); // 只有低預算
    } else if (budget === "Medium") {
      matchBudget = (String(d.cluster) === "2" || String(d.cluster) === "1" || String(d.cluster) === "4"); // 中預算包含低
    } else {
      matchBudget = true; // 高預算什麼都能去
    }

    // 3. 安全過濾 (全相容邏輯)
    let matchSafety = false;
    if (safety === "重視安全 (Safety First)") {
      matchSafety = (String(d.cluster) !== "4"); // 排除一線大城風險
    } else {
      matchSafety = true; // 可接受風險的人什麼都能去
    }

    return matchCategory && matchBudget && matchSafety;
  });
}
